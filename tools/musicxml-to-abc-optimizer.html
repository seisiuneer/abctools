MusicXML to ABC Optimizer

  • Paste ABC notation into the input area, open one or more .abc or .txt files, or send tunes directly from the ABC Transcription Tools.
  • Select any of the four optimization options. The Process ABC button is available only when the input contains an X: tune and at least one option is selected.
  • Shorten excessively long sustained notes removes only ties crossing measure lines when retaining the next tied segment would exceed the selected duration. Each tune uses its own Q:, M:, and L: values. When a tie is removed, any accidental required to preserve the sustained pitch is written explicitly on the newly re-attacked note.
  • Consolidate percussion voices for General MIDI playback and drum noteheads detects voices declared with perc or clef=perc and converts them using the selected method. The method selector is available only while this option is enabled.
  • Perc voice with percmap/drummap is the default method. It keeps converted voices as percussion voices, assigns every mapped General MIDI drum instrument in the tune one shared notation pitch, and generates matching tune-level %%percmap and %%MIDI drummap tables before all V: voice declarations, preceded by an % Injected percmap and drummap comment. Supported notehead values are retained in the %%percmap entries, and explicit accidentals prevent the tune key from changing the drum mapping. Previously optimized percussion tunes can be processed again without changing their drum mappings.
  • Treble voice with transpose=-24 (legacy) reproduces the optimizer's original conversion method. It changes percussion voices to treble voices, writes their mapped General MIDI pitches two octaves higher on the staff, and adds transpose=-24 so playback returns to the intended drum numbers. This method does not generate consolidated %%percmap or %%MIDI drummap tables and does not preserve imported drum noteheads.
  • For either method, if a percussion voice contains any usable I:percmap directives, those mappings take priority and %%MIDI drummap directives are fallback data. Both I:percmap <source-note> <GM-number> [notehead] and MusicXML-converter mappings such as I:percmap ^g g 42 x are recognized. The percmap/drummap method also retains the final supported notehead value.
  • General MIDI Level 2 percussion numbers 27 through 87 are accepted. Notes whose selected mapping is outside that range, and percussion notes with no selected mapping, are replaced with rests of the same duration and reported in the status area.
  • Both methods remove MIDI channel 10, old program and percussion-map directives, stafflines=1, and old transpose values before creating their replacement output. %%MIDI program 128 is added for General MIDI drum playback.
  • Existing nm= and snm= values are always removed from each percussion voice declaration. If that declaration originally had an nm= value, it receives nm="Drums". In that case, it also receives snm="Drums" when another non-percussion voice in the same tune uses an snm= value. No name is added to a percussion declaration that did not originally contain nm=.
  • Inject a soundfont command into each tune is enabled by default with MScore selected. It removes any existing %soundfont or %abcjs_soundfont command in the tune and adds the selected %soundfont <id> command near the beginning of the tune, after the generated staff-width and reverb directives when those directives are present.
  • Reformat music to a fixed number of measures per staff runs after the note and percussion operations. Choose from 1 through 8 measures per staff. Existing numeric end-of-line comments such as % 4 are removed, and each newly formatted voice line receives a comment showing its last measure number. Measure numbering starts at 1 independently for each voice.
  • Every processed tune receives %%staffwidth 1200 immediately after its first header K: field unless a %%staffwidth directive is already present in that tune. If the tune has no existing %reverb line or previously injected commented reverb setting, a commented chamber reverb setting is added immediately after the %%staffwidth directive.
  • When percussion consolidation is enabled, the processed tune also receives an %abcjs_render_params percussion volume scale and percussion-release setting before its voice declarations so volume can be adjusted and hi-hats and selected cymbals ring for appropriate lengths.
  • Operations run in this order: long-note shortening, percussion remapping, measure reformatting, soundfont injection, and final tune-level rendering directives.
  • After processing, the status area shows aggregate counts. Use Show Processing Details Report to see the tune index, title, and tune-by-tune counts.
  • Use the output buttons to copy, save, or send the processed ABC directly to the ABC Transcription Tools.
