Instructions
Getting Started
  1. Click Connect MIDI and allow browser MIDI access if requested.
  2. Choose your MIDI Input and Input Channel.
  3. Select one of the six instrument voices: Air, Reed, Solo, Brass, Pad, or Lead.
  4. Choose the Expression Source that matches your WARBL setup. This is a global setting for the entire tool and defaults to Note Velocity. You can switch it to Breath (CC2), Expression (CC11), or Volume (CC7) if preferred.
  5. Adjust tone, brightness, attack, release, reverb, noise/texture, and bend scaling to taste, then play your WARBL and watch the live status and MIDI log.
Instrument Voices
Air is the cleanest and usually the best starting point for checking pitch response.
Reed emphasizes odd harmonics for a reed-like tone.
Solo is a smooth, expressive solo sound with a sax-like character and works especially well for legato playing.
Brass adds stronger buzz and bite.
Pad uses a softer, chorused pad-like tone.
Lead is a brighter, more electronic monophonic lead.
Expression Source
Expression Source is a global setting for the entire tool and is not stored separately for each patch.
Breath (CC2) follows MIDI CC2 and controls playback level.
Expression (CC11) follows MIDI CC11 and controls playback level.
Volume (CC7) follows MIDI CC7 and controls playback level.
Note Velocity uses note-on velocity as the playback level.
Fixed Full Volume ignores incoming expression and plays at a constant level.
Pitch Bend and Portamento
Pitch Bend Range is a global setting for the entire tool, like the MIDI input channel.
The default is ±2 semitones, which is the most common default WARBL setup.
Bend Up Scale and Bend Down Scale let you make upward and downward bends feel different without changing the WARBL itself.
When Enable Portamento is turned on, the synth glides from one note to the next over the selected Portamento Time. It is off by default for all patches.
The new Solo patch is especially effective with a little portamento and expression control if you want a more lyrical, connected sound.
Reset Buttons
Reset Current Preset to Defaults restores only the currently selected voice to its default settings. Global settings such as Expression Source, MIDI Input, Input Channel, and Pitch Bend Range are not changed.
Reset All Presets to Defaults clears saved changes for all voices and restores the current voice to its default settings. Global settings such as Expression Source, MIDI Input, Input Channel, and Pitch Bend Range are not changed.
Saving and Loading Settings Files
WARBL Sound Lab lets you save your complete setup—including all preset settings, per-preset adjustments, and global options—to a file, then load it later or share it with others.
  • Export Settings:
    Click Export Settings to download a file containing all preset settings, per-preset adjustments, and global configuration.
  • Import Settings:
    Click Import Settings and select a previously saved file to restore your sounds and configuration.
  • What gets saved in the file:
    • All preset parameters (tone, brightness, texture, vibrato, portamento, reverb, etc.)
    • Preset-specific adjustments and overrides
    • Expression mode and pitch bend range
    • MIDI logging, meter, and performance options
  • What does not get saved in the file:
    • The selected MIDI input device
    • Browser- or hardware-specific connection state
WARBL Configuration Tool
The WARBL Configuration Tool button in the upper-left corner of the page opens the official WARBL Configuration Tool in a new browser tab.
This allows you to adjust fingering patterns, pressure sensitivity, MIDI output behavior, and pitch bend settings while using WARBL Sound Lab.
The configuration tool runs separately from this synth, so you can switch between the two browser tabs as needed.
Recommended Starting Setup
For initial testing, try Air with Note Velocity, bend range at 2 semitones, both bend scales at 1.00, and portamento turned off.
If you want a smoother, more expressive solo tone, try Solo with breath or expression control and a small amount of portamento.
Once the basic note response feels right, add brightness-follow, reverb, and optional portamento.
Lower CPU Use Mode
Lower CPU use mode reduces processor usage by simplifying parts of the internal synth voice while leaving the current reverb system in place.
This can help on slower computers, when running many browser tabs, or if you hear glitches, crackles, or delayed response.
When enabled, some patches may sound a little less rich or animated, especially the more layered sounds such as Pad, but the tool should remain more efficient overall.
You can further reduce CPU usage by disabling the output level meter and the MIDI stream logging.
⚠️ Keep This Tab or Window Visible While Playing
WARBL Sound Lab must remain visible on screen while you are playing.
All modern browsers (Chrome, Safari, Firefox, etc.) automatically reduce performance for content that is not visible to conserve CPU and battery. This includes:
  • Tabs that are not currently active
  • Windows that are fully covered by other windows
This behavior cannot be disabled by the tool.
When WARBL Sound Lab is not visible, incoming MIDI data from the WARBL (breath, expression, pitch bend, etc.) will not be processed smoothly.
This will cause:
  • Reduced responsiveness
  • Stepped or uneven dynamics and pitch
  • Audio glitches, clicks, or pops
As soon as the tab or window becomes visible again, normal performance will resume.
Important: This limitation affects all web-based audio tools.
Best experience: Keep WARBL Sound Lab in the foreground, or if using a separate window, make sure some portion of it remains visible at all times.
Troubleshooting
If you do not hear sound:
• Make sure MIDI access was granted.
• Verify the correct MIDI device is selected.
• Check that the correct input channel is selected.
• Try clicking All Notes Off if a note gets stuck.
If you hear pops or other glitches:
• Turn on Lower CPU use mode.
• Disable the output level meter and the MIDI stream logging.
• Close other browser tabs or programs that may be using CPU resources.
Browser Compatibility
This tool requires a browser that supports the Web MIDI API.
If your browser does not support WebMIDI, MIDI devices will not appear, and the tool will not function.
Recommended browsers:
• Google Chrome (Windows, macOS, Linux)
• Microsoft Edge (Windows, macOS)
• Opera (desktop versions)
• Chromium-based browsers that support WebMIDI
Not currently supported:
• Safari (macOS and iOS)
• Firefox (not officially supported; may work inconsistently)
• Web MIDI Browser app for iOS
• Most mobile browsers
